Projects

Buns on the Run

The first website that I created was for a baking blog called Buns on the Run. HTML and CSS were utilized in the creation of this site along with help of Dreamweaver as an IDE. I designed the entire site, including their logo which they still currently use on their YouTube channel.

While initially prototyping the site, I learned early on of the importance of Dreamweaver's templates in creating a multi-page website. The templates allowed me modify globally-shared code such as the headers and footers, without having to do it manually for every single page on the site.

I went through several rounds of prototyping a design which I felt was comfortable to look at; playing with colors and rearranging the layout until I was satisfied. A challenge I faced was to determine how to arrange the steps of each recipe to go alongside each of the images on the individual recipe pages. I felt that the best way to display this information to the users would be to implement a pure CSS slideshow of the images and number them to correspond with the step they would be when following the recipe.

My favorite part of this project is the CSS on the Recipes page. The customer had different categories (pet recipes, main dish and desserts) of recipes that they wanted featured on this page. I put three images for each category and as you hovered over them, CSS is then used to display the category name for that corresponding image. I was initially looking at a javascript solution but was surprised I could do it with CSS alone. The power of CSS is amazing!

Dimes Times

Dimes Times was another site created for a friend of mine who was interested in having a site designed to showcase his artwork and the books that he has written.

Again, this site was built with basic HTML and CSS. Later, I revisited the project in order to practice redesigning a website that was currently built for desktop screens into a website that would be responsive using media queries for mobile devices.

I used an HTML table to show the books that were on sale and to provide links if the users were interested in purchasing them. The table design became difficult when making the site responsive but it also taught me a lot about how to manipulate elements while trying to get the perfect design for a mobile site.

I experimented with the color palette for the site to convey a relaxed and mellow mood (based on the personality of the owner). My decision to make the entire site grayscale accomplished this. I felt that it would be best to make all of the images grayscale using CSS but also allowing the user the option to see the true representation of the artist’s work by hovering over the images in order to view them in color.

In the end, I actually believe that the mobile site looked a lot better than the original desktop site.

Yama Sushi Bar

The Yama Sushi Bar website, as well as the logo used on the site, was created as a school project for a Dreamweaver class. We were told to start with a basic template and to add in our own CSS to create our own design. One of the main topics discussed in the class, which we had to implement in this project, was to keep in mind that the site would have to be able to work on mobile devices.

The most difficult part of this project was the menu page. I really liked how it was designed for desktops with all of the images of the food surrounding the menu but I was really questioning how I would keep this design for mobile devices. I came to the conclusion that I wouldn’t be able to show as many images and still have the content be easily accessible. I decided that removing the images would be the best choice and then I could have the menu fill the width of the page. I didn't know it at the time but my train of thought illustrated the web design concept of 'graceful degradation'.

I was really happy with the outcome of this project on desktops as well as on mobile devices.
